# Memory Management Skill

このスキルは、ユーザーとの会話や作業過程で得られた情報を
適切なディレクトリに保存・整理するための手順を提供する。

このスキルは以下のような状況で使用する。

- ユーザーに関する長期的に有用な情報が判明したとき
- 継続的に扱う話題や関心テーマが見つかったとき
- 調査や解析の中間結果が大きく、会話コンテキストに直接載せるべきでないとき
- 作業途中の整理メモを保存したいとき
- ユーザーに提示する完成した成果物を保存するとき

---

# ディレクトリの意味

## /memories/profile/

ユーザーの長期的に再利用価値の高い情報を保存する。

例:

- 好きな技術
- 作業スタイル
- 長期的な目標
- よく使うツール
- 好み
- 継続的な前提条件

特徴:

- 変更頻度は低い
- 長期的に再利用される


---

## /memories/topics/

ユーザーが最近継続して扱っている話題や関心領域を保存する。

例:

- 最近調査しているテーマ
- 継続して議論しているトピック
- 未解決の問題

特徴:

- profileより更新頻度が高い
- 一時的だが継続する可能性がある

---

## /tmp/

大きな中間データや一時的な取得データを保存する。

例:

- Webページ全文
- 検索結果の一覧
- スクレイピング結果

特徴:

- コンテキスト節約のための退避場所
- 必要な部分だけ読み取って使用する


---

## /artifacts/

ユーザーに提示する完成した成果物を保存する。

例:

- レポート
- まとめ
- ドキュメント
- 設計メモ

特徴:

- 外部向け
- 完成版
- meta は PGVectorStore に保存されるため、検索は artifact_search → raw_path の順


---

# 保存判断ルール

情報を保存する前に次の判断を行う。

1. 長期的に役立つユーザー情報か
   → `/memories/profile/`

2. 最近の継続話題か
   → `/memories/topics/`

3. 大きな取得データか
   → `/tmp/`

4. 完成した成果物か
   → `/artifacts/`

---

# 推奨ワークフロー

1. 情報を生成または取得する
2. サイズが大きい場合 `/tmp/` に保存
3. 必要な部分だけ読み取る
4. 作業途中の整理は `/workspace/`
5. 有用な知識は `/memories/` に保存
6. 最終成果物は `/artifacts/` に保存（meta は PGVectorStore に保存）

---

# ファイル探索

保存された情報を探す場合は次の順序を推奨する。

1. `artifact_search` でメタを検索する
2. raw_path を辿り raw.json を読む

---

# 更新ルール

既存の情報が存在する場合:

- 同じ情報なら更新する
- 重複保存を避ける
- 必要に応じて要約して保存する

---

# 重要原則

- 大きなデータは直接会話に含めない
- ファイルシステムを利用してコンテキストを節約する
- 再利用価値の高い情報だけを `/memories/` に保存する
