# Memory Management Skill

このスキルは、ユーザーとの会話や作業過程で得られた情報を
LangMem とファイルシステムへ適切に保存・整理するための手順を提供する。

このスキルは以下のような状況で使用する。

- ユーザーに関する長期的に有用な情報が判明したとき
- 継続的に扱う話題や関心テーマが見つかったとき
- 調査や解析の中間結果が大きく、会話コンテキストに直接載せるべきでないとき
- 作業途中の整理メモを保存したいとき
- ユーザーに提示する完成した成果物を保存するとき

---

# 保存先の意味

## LangMem Long-Term Memory

ユーザーの長期的に再利用価値の高い情報や継続話題は、
ファイルとしてではなく LangMem に保存される。

例:

- 好きな技術
- 作業スタイル
- 長期的な目標
- よく使うツール
- 好み
- 継続的な前提条件
- 最近継続して扱っている話題
- 関心領域
- 未解決の問題

特徴:

- main agent 実行前に検索され prompt に注入される
- main agent 実行後に ReflectionExecutor で非同期保存される
- `/memories/` をファイルとして read/write しない

---

## /tmp/

大きな中間データや一時的な取得データを保存する。

例:

- Webページ全文
- 検索結果の一覧
- スクレイピング結果

特徴:

- コンテキスト節約のための退避場所
- 必要な部分だけ読み取って使用する

---

## /artifacts/

ユーザーに提示する完成した成果物を保存する。

例:

- レポート
- まとめ
- ドキュメント
- 設計メモ

特徴:

- 外部向け
- 完成版
- meta は PGVectorStore に保存されるため、検索は artifact_search → raw_path の順

---

# 保存判断ルール

情報を保存する前に次の判断を行う。

1. 長期的に役立つユーザー情報か
   → LangMem long-term memory

2. 最近の継続話題か
   → LangMem long-term memory

3. 大きな取得データか
   → `/tmp/`

4. 完成した成果物か
   → `/artifacts/`

---

# 推奨ワークフロー

1. 情報を生成または取得する
2. サイズが大きい場合 `/tmp/` に保存
3. 必要な部分だけ読み取る
4. 作業途中の整理は `/workspace/`
5. 有用な知識は LangMem に保存される前提で扱う
6. 最終成果物は `/artifacts/` に保存（meta は PGVectorStore に保存）

---

# ファイル探索

保存された情報を探す場合は次の順序を推奨する。

1. `artifact_search` でメタを検索する
2. raw_path を辿り raw.json を読む

---

# 重要原則

- 大きなデータは直接会話に含めない
- ファイルシステムを利用してコンテキストを節約する
- 再利用価値の高い情報だけを LangMem の long-term memory に残す
